曾經做過哪些軟體工程技術相關的學習？若有作品請分享給我們。

  • 在軟體工程技術相關的學習上，我主要針對資料、數據分析為主，偶爾會學習資料庫、ETL和API開發以助於資料清洗、演算法上的佈署，以下為相關具體的學習與實踐經驗：
    • 資料、數據分析技術：

      常用Pandas、NumPy 等資料處理與分析的套件，進行資料清理、處理和轉換；使用 Matplotlib進行資料視覺化，來瞭解資料趨勢。 模型上因工作需求目前主要針對在時間序列上的分析，包括使用 Scikit-learn、Statsmodels等套件進行趨勢上模型的構建，來預測設備老化狀態。

    • 資料庫、ETL技術:

      為了後續的佈署整體的資料流，需接觸些相對應的資料庫相關知識、資料表的建立，有助於後續在撈取資料跟做資料清洗，使整體的flow更加順暢，透過Python script 讓數據管線的自動化處理。

    • FastAPI 串接與後端開發:

      目前使用 FastAPI 框架進行 API 開發，已實作過幾個 RESTful API 專案，串接資料庫如 PostgreSQL，並使用SQLAlchemy 進行 ORM 操作。 在專案中，開發了多個 API 端點，有效處理不同的 HTTP 請求（如 GET、POST、PUT、DELETE），讓前端使用者的相對應需求與演算法上能夠有良好的橋樑，並回傳相對應的結果，此外熟悉如何使用 FastAPI 進行驗證和錯誤處理，確保 API 的安全性與穩定性。

    • Poetry 環境建置與打包：

      針對環境的建立相對應學習上，因應專案需求，目前使用Poetry 來進行 Python 專案的依賴管理與打包，讓專案的維護更簡單，目前能熟練處理專案環境建置，並解使用Poetry對於每項function做出打包，讓彼此之間能夠同時共用。

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

  • 事件：針對某次演算法轉譯專案時，要把matlab版本轉譯成python版，受到時間因素影響，需盡快完整轉譯、單元測試以及壓力測試，由於原始版本並未留下相關的文件說明，因此瞭解此算法花費了不少時間，轉譯的過程中，發現轉譯結果的偏差相當大，當時壓力十分大，怕無法完成此項工作。
  • 處理方式：
    • 針對這次事件，我是先冷靜下來，逐一的分析問題，針對原始matlab code做出逐行的拆解，確認每次的運算內容，並和同事討論可能出現的問題，最後發現是模型的建模上，由於套件因素，讓取樣的sample數不同，導致結果趨勢上有大偏差。
    • 最終我也如期的完成轉譯，讓程式上的結果趨勢符合，也是這次經驗讓我應對突發狀況的能力有所精進。
